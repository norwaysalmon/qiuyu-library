## Phase 4：基础体验升级（近期）

### 4.1 外部网站快捷链接 ⭐⭐⭐ ★☆☆

**描述：** 在图书馆主界面引入常用网站链接，以大图标形式展示，方便用户快速跳转。

**具体需求：**
- [ ] 设计链接卡片组件（大图标 + 网站名称）
- [ ] 初始链接池：Bilibili、YouTube、Google、GitHub、Twitter/X 等
- [ ] 支持用户自定义添加/删除链接
- [ ] 响应式布局（桌面端网格 / 移动端横向滚动）
- [ ] 链接数据持久化（localStorage 或 R2）

**分工：**
- 🎨 **李沐然**：卡片视觉设计、hover 动效、图标风格（像素风/西幻风）
- 🛠️ **姐姐**：数据结构设计、localStorage 持久化逻辑、R2 同步（如需要）

**所需资源：** 无外部 API，纯前端

### 4.2 西幻 + 像素风格 UI 改造（第一阶段） ⭐⭐⭐ ★★☆

**描述：** 在保持现有简洁骨架的基础上，逐步注入西幻魔法感 + 像素风元素。

- [ ] **书架/魔法元素装饰**：侧边栏或背景加入微妙的魔法光晕、符文边框
- [ ] **页面过渡动画**：翻书效果或魔法传送门效果
- [ ] **像素风 favicon**：图书馆的像素风小图标

**分工：**
- 🎨 **李沐然**：所有视觉设计、CSS 动画、过渡效果
- 🖼️ **GPT-5.4 Image 2**（姐姐调用）：生成像素风图标素材、羊皮纸纹理、符文装饰等
- 🖼️ **Cloudflare Flux/SDXL**（姐姐调用）：生成背景氛围图、装饰元素
- 🛠️ **姐姐**：素材集成、性能优化、部署

**所需资源：**
- OpenRouter API（GPT-5.4 Image 2）— 图像生成
- Cloudflare Workers AI（Flux）— 免费图像生成备选
- 像素风字体推荐：`Press Start 2P`、`Silkscreen`、`VT323`

## Phase 5：核心功能扩展（中期）

### 5.1 访客系统 ⭐⭐ ★★★

**描述：** 从当前的单管理员密码模式，扩展为支持访客注册、登录、权限管理的完整用户系统。

**具体需求：**
- [ ] **访客注册流程**：阅读守则 → 签名确认 → 魔法羽毛笔动画 → 生成访客 ID（格式如 `visitor_XXXX`）
- [ ] **访客登录**：访客 ID + 自设密码
- [ ] **权限分级**：
  - 管理员（祈鸳）：全部权限
  - 访客：浏览图书馆、使用公共功能
  - （未来）VIP 访客：可上传文件、自定义小人等
- [ ] **访客资料存储**：R2（JSON 文件）或 Cloudflare KV
- [ ] **访客界面**：与管理员界面有所区分（隐藏管理功能）
- [ ] **会话管理**：登录态保持（sessionStorage / cookie）

**分工：**
- 🎨 **李沐然**：注册/登录界面视觉、签名动画、访客 ID 生成动画
- 🛠️ **姐姐**：后端 API（Worker 扩展）、用户数据存储、权限逻辑、安全设计

**所需资源：**
- Cloudflare Worker（扩展现有 worker.js）
- Cloudflare R2 或 KV（用户数据存储）
- Web Crypto API（密码哈希，已有 SHA-256 实现）

**技术方案（草案）：**
```
Worker 新增路由：
POST /api/auth/register  → 注册新访客
POST /api/auth/login     → 访客登录
GET  /api/auth/me        → 获取当前用户信息
PATCH /api/auth/me       → 修改用户资料

R2 存储结构：
users/
  ├── admin.json          → 管理员信息
  └── visitors/
      ├── visitor_a1b2.json
      └── visitor_c3d4.json
```

### 5.2 AI 聊天界面（沈秋月） ⭐⭐ ★★☆

**描述：** 在图书馆内嵌入一个 AI 对话窗口，让沈秋月可以与访客进行实时互动。

**具体需求：**
- [ ] **对话界面**：右下角浮动按钮 → 点击展开聊天窗口（类似客服 widget）
- [ ] **模型接入**：DeepSeek V4 Pro（通过 OpenRouter API）
- [ ] **系统提示词**：加载沈秋月的人设（IDENTITY.md + SOUL.md 精简版）
- [ ] **对话历史**：单次会话内保持上下文，关闭窗口后清除
- [ ] **流式输出**：打字机效果，逐字显示回复
- [ ] **视觉风格**：西幻风聊天框（羊皮纸背景、魔法墨水字体）
- [ ] **（未来）像素小人联动**：聊天时右下角的像素小人有对应动画反应

**分工：**
- 🎨 **李沐然**：聊天窗口视觉设计、展开/收起动效、消息气泡样式
- 🛠️ **姐姐**：OpenRouter API 集成、流式输出、系统提示词管理、错误处理

**所需资源：**
- OpenRouter API Key（已有）
- 模型：`deepseek/deepseek-v4-pro`（$3/M input tokens）
- 注意：API Key 不能暴露在前端代码中，需要通过 Worker 中转

**技术方案（草案）：**
```
前端 → Worker 中转 → OpenRouter API
POST /api/chat
Body: { messages: [...], system_prompt: "..." }
Response: SSE stream (text/event-stream)

安全措施：
- API Key 仅存储在 Worker 环境变量中
- 限流：每访客每分钟最多 10 条消息
- 内容过滤：Llama Guard 3（Cloudflare Workers AI，免费）
```

## Phase 6：内容生态（远期）

### 6.1 新闻聚合模块 ⭐ ★★☆

**描述：** 在图书馆内嵌入新闻阅读区域，展示高热度头条新闻。

**具体需求：**
- [ ] **新闻源接入**：RSS 订阅或 API（如 RSSHub、NewsAPI）
- [ ] **分类展示**：科技、财经、国际、娱乐等分类标签
- [ ] **卡片式布局**：标题 + 摘要 + 来源 + 时间
- [ ] **点击跳转**：在新标签页打开原文
- [ ] **自动更新**：每小时或每日定时拉取最新新闻

**分工：**
- 🎨 **李沐然**：新闻卡片视觉、分类标签样式
- 🛠️ **姐姐**：RSS 解析、数据缓存（Worker + R2）、定时更新（Cron Trigger）

**所需资源：**
- RSSHub（免费，自建或公共实例）
- Cloudflare Worker Cron Trigger（免费，每天最多 3 次）

**潜在问题：**
- RSS 源的稳定性和时效性
- 部分网站反爬，需要备用方案
- 新闻内容的版权合规性

### 6.2 音乐播放器 ⭐ ★★☆

**描述：** 在网页角落嵌入一个迷你音乐播放器，支持用户自定义音乐源。

**具体需求：**
- [ ] **播放器 UI**：迷你悬浮条（播放/暂停、进度条、音量、歌曲名）
- [ ] **音乐源支持**：
  - YouTube URL → 提取音频（需后端处理，或使用 iframe 嵌入）
  - 用户直接提供音频 URL（MP3/OGG）
  - （未来）内置白名单音乐库
- [ ] **播放列表**：用户可添加/删除/排序歌曲
- [ ] **播放列表持久化**：localStorage 或 R2
- [ ] **视觉风格**：复古唱片机或魔法八音盒风格

**分工：**
- 🎨 **李沐然**：播放器 UI 设计、播放动效（唱片旋转等）
- 🛠️ **姐姐**：音频播放逻辑、YouTube iframe 集成、播放列表管理

**所需资源：**
- YouTube IFrame API（免费）
- HTML5 Audio API（原生）

**技术方案（草案）：**
```
方案 A（简单）：YouTube IFrame 嵌入，隐藏视频只留音频控制
方案 B（完整）：后端提取 YouTube 音频流 → 前端 Audio API 播放
  - 需要额外的后端服务（如 ytdl-core）
  - 版权风险较高
方案 C（推荐）：支持用户粘贴任意音频 URL + YouTube iframe 双模式
```

### 7.2 用户像素小人（捏脸系统） ⭐ ★★★★

**描述：** 让访客可以创建自己的像素小人形象，并在图书馆底部栏中行走。

**具体需求：**
- [ ] **捏脸系统**：
  - 发型、发色、肤色、服装、配饰等可自定义选项
  - 实时预览
  - 保存为用户资料的一部分
- [ ] **行走系统**：像素小人在网页底部栏随机行走（Terraria 风格）
  - 行走、站立、吃东西、拿出火把等随机动作
  - 碰到页面边缘转向
- [ ] **多小人共存**：页面上同时显示多个用户的小人

**分工：**
- 🖼️ **GPT-5.4 Image 2 / Flux**：生成像素服装/配饰素材库
- 🎨 **李沐然**：捏脸 UI、Canvas 行走系统、动画状态机
- 🛠️ **姐姐**：后端存储（用户小人数据）、数据同步逻辑

**技术方案（草案）：**
```
Canvas 底部栏（固定定位，高度 ~64px）：
┌─────────────────────────────────────────────┐
│ 🧙‍♀️ 🐺    🧝‍♀️        🧛‍♂️    🧙‍♂️        🐱  │
│ ═════════════════════════════════════════════ │
└─────────────────────────────────────────────┘

小人数据结构：
{
  "visitor_id": "visitor_a1b2",
  "sprite": {
    "hair": "long_brown",
    "skin": "fair",
    "outfit": "witch_robe",
    "accessory": "none"
  },
  "position": { "x": 120, "direction": "right" },
  "action": "walking"  // walking, standing, eating, torch
}
```

### 7.3 像素 NPC 系统 ⭐ ★★★

**描述：** 在图书馆中随机生成各种西幻主题的像素 NPC，增加页面活力。

**具体需求：**
- [ ] **NPC 类型**：狼人、女巫、吸血鬼、精灵、矮人、猫等
- [ ] **随机行为**：
  - 从页面边缘探出头 → 停留几秒 → 消失
  - 从一侧走到另一侧 → 偶尔停下来互动
  - 在某个角落出现 → 做个动作 → 离开
- [ ] **互动反馈**：鼠标悬停或点击时显示对话气泡（随机台词）
- [ ] **出现频率**：每隔 30-60 秒随机生成一个 NPC

**分工：**
- 🖼️ **GPT-5.4 Image 2 / Flux**：生成各类 NPC 的像素 Sprite
- 🎨 **李沐然**：NPC 行为状态机、出现/消失动画、对话气泡
- 🛠️ **姐姐**：NPC 台词库编写、出现频率调优

### 7.4 多用户实时同步（终极愿景） ⭐ ★★★★★

**描述：** 当多个用户同时浏览图书馆时，其他用户的小人会实时出现在页面中。

**具体需求：**
- [ ] **实时通信**：WebSocket 或 Server-Sent Events
- [ ] **在线状态**：显示当前在线人数
- [ ] **小人同步**：其他用户的小人位置和动作实时更新
- [ ] **互动**：用户小人之间可以"打招呼"

**技术方案（草案）：**
```
方案 A：Cloudflare Durable Objects（WebSocket）
  - 优点：与现有 Cloudflare 生态无缝集成
  - 缺点：Durable Objects 需要付费计划

方案 B：Supabase Realtime（免费额度）
  - 优点：免费、开箱即用
  - 缺点：引入第三方依赖

方案 C：轮询（最简单但体验最差）
  - 不推荐，仅作 fallback
```

> ⚠️ 此功能暂不实现，仅作技术预研。等 Phase 7.1-7.3 完成后再评估。
